## 1. Background & Problem Statement

CVS pharmacists currently spend **20–40 minutes per patient** manually generating care plans. These care plans are:

- Required for **Medicare reimbursement** and **pharma compliance**
- Currently backlogged due to staff shortages
- Generated by reviewing patient medical history and producing a structured document

This system automates care plan generation using an LLM, triggered by a web form filled out by CVS medical staff.

> **Users:** CVS medical workers (pharmacists, medical assistants). Patients do not interact with this system. Generated care plans are printed and handed to patients.

## 6. Business Logic: Duplicate Detection Rules

### Provider

| Scenario | Action | Reason |
|---|---|---|
| NPI match + name match | ✅ Reuse existing provider | Normal case |
| NPI match + name **mismatch** | ❌ **ERROR — block submission** | NPI is a national unique license number |

### Patient

| Scenario | Action | Reason |
|---|---|---|
| MRN match + name & DOB match | ✅ Reuse existing patient | Normal case |
| MRN match + name or DOB **mismatch** | ⚠️ **WARNING — user can confirm and continue** | Possible data entry error |
| Name + DOB match + MRN **mismatch** | ⚠️ **WARNING — user can confirm and continue** | Possibly the same person |

### Order

| Scenario | Action | Reason |
|---|---|---|
| Same patient + same drug + **same day** | ❌ **ERROR — block submission** | Definitely a duplicate submission |
| Same patient + same drug + **different day** | ⚠️ **WARNING — user can confirm and continue** | Possibly a refill |

---

## 7. Error Handling Philosophy

Three categories of "abnormal" situations:

| Type | HTTP Status | Example | Behavior |
|---|---|---|---|
| **Validation Error** | 400 | NPI is not 10 digits | Blocked, must fix input |
| **Business Block (ERROR)** | 409 | NPI conflict with different provider name | Blocked, must resolve before continuing |
| **Business Warning** | 200 + `warnings` field | Possible duplicate patient | User sees warning, can confirm and continue |

All errors return a **unified JSON format**:
```json
{
  "type": "block | warning | validation",
  "code": "DUPLICATE_NPI",
  "message": "This NPI already exists with a different provider name.",
  "detail": {}
}
```

## 8. Care Plan Output Format

Every generated care plan **must** include these four sections:

```
1. Problem List / Drug Therapy Problems (DTPs)
2. Goals (SMART format)
3. Pharmacist Interventions / Plan
4. Monitoring Plan & Lab Schedule
```

## 10. Non-Functional Requirements

- Every input validated before hitting the database
- No PHI (Protected Health Information) exposed in error messages or logs
- LLM failures must not crash the system — retry up to 3 times, mark as `failed` if all retries exhausted
- Project must run end-to-end with a single command (`docker compose up`)
- Critical business logic covered by automated tests
